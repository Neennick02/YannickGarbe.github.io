Dungeon Danger

About

Dungeon Danger is a action adventure game in which you explore an ancient temple filled with dungerous enemies and puzzles.
Activate all stair engines to open up the way to the final boss room.

Project Info

Engine: Unity
Development time: 10 weeks
Team size: 1 developer

Introduction

Dungeon Danger is a 3D action adventure game inspired by retro games like The Legend Of Zelda Ocarina Of Time.
Explore the dungeon, fight foes and solve puzzles to confront the final boss and free the temple of its curse.

Gameplay

Control the player with keyboard & mouse or controller. Jump, dodge, block and attack your way through numerous rooms.

Interact with objects to solve puzzles, open doors and save your current progress.
The action button in the top right gives a prompt if interactable objects are in range.

Collect coins found in pots to save progress at save statues.
Saving is available at different statues for different amounts of coins.

Systems I worked on.

  • Player Movement System – Character controller based movement.

  • Player Interaction System – Interact with objects.

  • Player Combat System – Animation Based Combat.

  • Enemy Targeting System – Enemies are added to a pool and sorted by distance.

  • Dynamic Music System – Music Changes during Combat.

  • Save System – Save Current Progress via PlayerPrefs

  • Enemy StateMachines And Behaviour –
    Basic Combat States that drive behaviour and animations.

  • Locked Doors System – Doors with requirements like:
    Kill All Enemies, Light All Torches or Light Torches in specific order.

  • Dialog System – A typewriter effect that can be sped up or skipped completely.

  • Cut Scene Systems – Shows Player What Changed After Puzzle is Solved

  • Boss Behaviour and Movmement – Spline based Movement and movement speed increases after damage.

What I've learned.

This was the first time I was able to work on a project for a longer time which gave me a lot of time to plan out idea's and make a detailed planning.

In my Game Design Document I described exactly what I wanted to make and in my Technical Design Document I described how I planned to work out some Systems.

During the making of this project I learned a lot about scope, making small games is easy but making a bigger game is harder than I thought.
All systems, mechanics and design choises need to scale with the entire project.

This was my first time designing puzzles and a map. I learned how to use the Unity Pro Builder package to build the map.
Having multiple playtests really helped me out with making the tutorial section clear for new players.

While making the Z targeting system and cutscene systems I learned a lot about Unity's Cinemachine package.

I also focussed a lot of time on keeping systems decoupled from each other to prevent bugs a keep a clean code base.

This project was part of a school assignment meant to teach me about freelance projects, this includes a fictional contract and a detailed planning.
