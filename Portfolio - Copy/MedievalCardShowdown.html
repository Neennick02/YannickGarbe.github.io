Medieval Cards Showdown

About

Medieval Cards Showdown is a turn based card game with 21 different cards.
Battle multiple opponents and use stronger cards in later rounds.

Project Info

Engine: Unity
Development time: 3 weeks
Team size: 1 developer

Introduction

Turn Based Cards is inspired by games like heartstone and slay the spire, spend mana on cards to attack, heal or defend.
Use mana to use cards with different costs and can last up to multiple

Fight 3 opponents with 21 different cards that consists of 3 types: Weapons, Spells and Potions.

Gameplay

Hover over cards to get a better look at the effects, left click to use card and mouse click to discard frm your hand.

Every round the mana counter increases by 1 which allows the player to use stronger cards.

Some cards have effects that last multiple rounds.
A card only has 1 instance in the deck, the deck gets shuffeled if all cards are grabbed.

Systems I worked on.

  • Turn Based System – Cards can only be selected during the player's turn.

  • Mana System – Increases every round, depletes when using a card.

  • Card System – ScriptableObjects with card data and images.

  • Enemy AI System – Enemy decides turn based on current health.

  • Game Loop – Check player and opponent health.

  • Card Hover Effect – Zoom in card on mouse hover.

  • Cards Effects – Card effects can last for multiple rounds.

  • Particle System Implementations – Instantiates particle system based on used card.

  • Scrolling Effect – Scrolls cards in deck scene and credit scene.
    Scrolls over time and can be controlled with mouse wheel.

  • Health Systems – Deals damage, heals and updates health bars.

  • Changing Opponent Systems – All enemies use the same animation controller but have different models and attack stats.

  • Animation Systems – Plays animations based on used cards.

What I've learned.

When I started working on this project I wasn't familiar with this game genre.

While working on the Game Design Document I thought a lot about the card designs to make them clear to read.

I had some issues with z fighting while programming the cards in the players hand, this was because all cards were attached to a spline that controlled the z positions.

The player and opponent anitmations have some issues with root motion animations because the animation clips don't loop correctly.


After working on the core gameplay loop I decided to add 3rd party assets to make the map and start scene more alive, this caused issues while trying to make a WEBGL build for itch.io.
This taught me about the limits for WEBGL file sizes.
